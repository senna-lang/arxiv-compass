## Background

`scripts/map.py` が生成するクラスタキーワードの品質が低い。原因はc-TF-IDFのデフォルト設定で論文特有の汎用語（"model", "method", "propose"等）が除去されないため、クラスタを特徴づけない語がキーワードとして上位に来てしまう。

対象ファイル: `scripts/map.py`（`CountVectorizer` と `BERTopic` の設定部分）

## Spec

3段階のアプローチを順番に実装する。

**プランA: 学術ストップワード追加（即効・ゼロコスト）**
- `CountVectorizer` に論文特有の汎用語リストを追加
- `min_df` を 2→3 に引き上げ
- 追加ライブラリ不要、処理時間変化なし

```python
ACADEMIC_STOPWORDS = [
    "model", "method", "approach", "propose", "proposed", "show", "result", "results",
    "paper", "work", "task", "performance", "training", "dataset", "data",
    "experiment", "experiments", "demonstrate", "achieve", "achieved", "using", "based",
    "learning", "neural", "deep", "large", "new", "existing", "different", "various",
]
vectorizer = CountVectorizer(
    stop_words=list(CountVectorizer(stop_words="english").get_stop_words()) + ACADEMIC_STOPWORDS,
    ngram_range=(1, 2),
    min_df=3,
)
```

**プランB: KeyBERTInspired representation（意味的再ランキング）**
- BERTopic同梱の `KeyBERTInspired` を `representation_model` に設定
- c-TF-IDF候補をEmbeddingの意味的近さで再ランキング
- 処理時間 +10〜20%、品質向上大

```python
from bertopic.representation import KeyBERTInspired
representation_model = KeyBERTInspired()
topic_model = BERTopic(..., representation_model=representation_model)
```

**プランC: Claude APIによる自然言語ラベリング（任意）**
- キーワード抽出はA+Bのまま
- `label` フィールドのみClaudeで生成（top3キーワード + 代表abstract3件 → 1行ラベル）
- `config.json` に `claude_labeling: true/false` フラグを追加
- コスト: Claude APIトークン従量課金

## Key Decisions

Decision: A→B の順で実装し、Cは別途判断。Rationale: AはリスクゼロでBの土台になる。CはAPIコストが発生するため効果確認後に検討。

Decision: Aのストップワードリストは `map.py` 内の定数として定義（config化しない）。Rationale: チューニングはコードレベルで管理した方がdiff管理が楽。

## Notes

- 検証方法: `npm run map`（1000件）で実行後、`data/map.json` の `keywords` と `label` を目視確認。`/map` ページのツールチップで視覚確認。
- プランBは `bertopic[representation]` が必要な場合があるが、通常インストール済み。
