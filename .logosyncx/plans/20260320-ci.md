## Background

現在このプロジェクトにはバッチ処理用のワークフロー（daily/weekly/monthly）のみ存在し、PR・push 時に実行される CI がない。コード品質の検証（型チェック、ビルド確認、テスト、lint）を自動化するため、CI ワークフローを新規作成する。

## Spec

### 作成ファイル: `.github/workflows/ci.yml`

- **トリガー**: `push` to `main` + PR targeting `main`
- **concurrency**: `ci-${{ github.ref }}` + `cancel-in-progress: true`

### 4 つの並列ジョブ

| Job | 内容 | 依存ツール |
|-----|------|-----------|
| `typecheck` | `npx tsc --noEmit` | Bun |
| `astro-build` | `bun run build` | Bun |
| `python-test` | `pytest scripts/tests/ --ignore=test_parse.py` | Python 3.11 + requirements.txt + pytest |
| `python-lint` | `ruff check scripts/` + `py_compile` | Python 3.11 + ruff |

### キャッシュ戦略

- Bun: `oven-sh/setup-bun@v2` の `cache: true`
- pip: `actions/setup-python@v5` の `cache: pip`

### 追加変更

- `requirements.txt` に `pytest` を追加

## Key Decisions

- Decision: `test_parse.py` を `--ignore` で除外。Rationale: 存在しない `parse.py` を import しており修正は別 issue。
- Decision: `bun test` ジョブは作らない。Rationale: フロントエンドのテストファイルが現時点で存在しない。
- Decision: `ruff.toml` は作らない。Rationale: まず既定ルールで実行し、必要に応じて後からカスタマイズ。
- Decision: python-lint は ruff + py_compile のみで ML 依存不要。Rationale: 高速（~20秒）に保つため。

## Notes

- `data/*.json` はリポジトリにコミット済みなので astro-build に必要なデータは揃っている
- 検証: feature ブランチで PR を作成し、4 ジョブが並列で走ることを確認する
