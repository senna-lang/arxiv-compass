## Background

`scripts/` 配下の Python スクリプト（fetch_daily.py: 335行, map.py: 427行, recommend.py: 409行）がモノリシック構造。`load_config()`, `_cosine_similarity()`, ratings読込など同一コードが3箇所に重複しており可読性が低い。単一責任・テスタブル性を意識した構造にリアーキする。

## Spec

### 新ディレクトリ構成

```
scripts/
  core/                          # 共通ユーティリティ
    config.py                    # load_config(), ROOT, CONFIG_PATH, JST
    similarity.py                # cosine_similarity(), mean_cosine_similarity()
    ratings.py                   # load_ratings() HTTP/ローカルフォールバック
    io.py                        # save_json(), append_jsonl()
    arxiv_client.py              # strip_version(), build_category_query()

  fetch_daily/                   # 日次収集パイプライン
    cli.py                       # argparse + main() オーケストレーション + modal_main
    fetch.py                     # fetch_recent_papers() - arXiv API
    dedup.py                     # load_seen_ids(), deduplicate()
    scoring.py                   # score_papers() - α-blend スコアリング

  map_pipeline/                  # クラスタリングパイプライン
    cli.py                       # argparse + main() + modal_main
    fetch.py                     # fetch_arxiv_papers() + pickle キャッシュ
    clustering.py                # ACADEMIC_STOPWORDS, build_bertopic_model()
    aggregation.py               # generate_label(), build_cluster_dict(), build_map_output()
    visualization.py             # generate_map_html() - datamapplot

  recommend/                     # レコメンドパイプライン
    cli.py                       # argparse + main() + modal_main
    cluster_ranking.py           # compute_instance_score(), compute_alpha(), rank_clusters()
    serendipity.py               # select_serendipity_papers()
    visualization.py             # regenerate_map_html()

  specter2.py                    # 変更なし
  modal_app.py                   # 変更なし
  benchmark.py                   # import先を更新
```

### CLIエントリポイント

`package.json` を `python -m scripts.fetch_daily` 等に更新。各パッケージの `__init__.py` で `from .cli import main` を re-export。

## Key Decisions

- Decision: `map_pipeline` 命名。Rationale: `map` はPython組み込み関数と衝突するリスク回避。
- Decision: 薄い cli.py パターン。Rationale: I/Oとオーケストレーションのみにし、ビジネスロジックを分離してテスタブル性確保。
- Decision: core/ → パイプラインの一方向依存のみ。Rationale: 循環参照防止。
- Decision: modal_app.py は据え置き。Rationale: Modal entrypoint routing の複雑化を避ける。
